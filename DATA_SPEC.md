# 데이터 명세서 (Data Spec)

> 목적: 본 PoC를 '장난감'이 아니라 **실데이터 전환 설계도**로 제시한다.
> 더미라도 "실제로는 어디서 오는가"를 명시해, 실데이터 교체 지점을 분명히 한다.

## 1. 산출물별 출처 매핑

| 항목 | 실데이터 출처(가정) | 데모 더미 생성 방식 | 교체 지점(파일) |
|---|---|---|---|
| 전이 카운트 | HR 시스템 인사이동 이력 테이블 (연도별 직급·근속 변화, **다년 누적**) | `generate_dummy.py`가 **2020~기준연도 이력 패널**을 시뮬레이션 — 매년 전이확률 가정값(`TRANSITION_ASSUMPTIONS`)으로 개인을 추적(이탈 퇴장 + 신규채용 보충). 전 연도를 풀링해 카운트 | `generate_dummy.py` |
| 초기 인력벡터 n(0) | 재직자 현황 스냅샷 (기준연도) | 패널에서 **기준연도 시점 재직자**만 집계(`get_base_year_headcount`) — 전 연도를 합산하지 않음. 초기 코호트는 `INITIAL_STATE_MIX` | `generate_dummy.py` |
| 상태 라벨 | 직급코드 + 근속연수 파생 | 직급 + 근속밴드로 상태 부여 | `data_pipeline.py` |
| 조정계수 | 외부 리서치 → 전문가 검토(델파이) | LangGraph 에이전트 web search 추출, 키 없으면 매핑 테이블 상수 | `research_agent.py` |
| 목표 필요인력 | 사업계획·조직설계 TO | 사이드바 수기 입력 | `app.py` 사이드바 |

## 2. 개인 단위 레코드 스키마

| 컬럼 | 타입 | 설명 |
|---|---|---|
| 사번 | str | 고유 식별자 |
| 직급 | str | 사원/대리/차장 |
| 근속연수 | int | 현 직급 기준 근속(년) |
| 부서 | str | 조직 단위 |
| 관측연도 | int | 전이 출발연도 (2020 ~ 기준연도-1). 1행 = 해당 연도의 인사이동 1건 |
| 현재상태 | str | 관측연도 시점 상태, 7개 중 하나 (직급+근속밴드) |
| 다음연도_상태 | str | 1년 후(관측연도+1) 상태 (전이 라벨, 이탈 포함) |

> 데이터는 단일 스냅샷이 아니라 **여러 해가 세로로 쌓인 이력 패널(long format)**이다.
> 실데이터에선 `현재상태`/`다음연도_상태`를 인사이동 이력에서 직급·근속 변화로 산출한다.

## 3. 상태 정의 (7개, duration-aware 준마르코프)

`사원_0-2년, 사원_2년+, 대리_0-2년, 대리_2년+, 차장_0-2년, 차장_2년+, 이탈(흡수상태)`

- 근속밴드를 상태에 넣어 무기억성(마르코프) 한계를 부분 보완.

## 4. 핵심 가정 (데모 스토리)

- 사원→대리 승진 **활발**, 대리→차장 승진 **정체**
- 차장 구간 이탈률 **높게** → 결과 차트에서 "차장 병목/누수"가 드러나도록 의도적으로 구조화

## 5. 고도화 로드맵

- 데이터 규모 충분 시: **계층적 풀링 / 베이지안(디리클레) 추정**으로 전이확률 안정화
- 상태 해상도 확장: `markov.py`의 `STATES`만 변경(직급/근속밴드 추가)
