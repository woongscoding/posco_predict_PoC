# Claude Code 프롬프트 — 선제적 인력 예측 PoC (데모용)

> 아래 전체를 Claude Code에 그대로 붙여넣으면 됩니다.
> 굵은 전제: **이건 실데이터 추정이 아니라 "파이프라인이 단계별로 돌아가는 걸 보여주는 데모"**입니다. 숫자 정확도보다 **과정의 시각화**와 **실데이터 교체 용이성**이 평가 포인트입니다.

---

## 0. 목표 한 줄

전략 컨설팅 데모용 **Streamlit 앱**을 만든다. 사용자가 "기준연도 2026 → 목표연도 2031"을 입력하면, 내부 인력데이터를 끌어와 전처리하고 **마르코프 모델링**으로 인력을 투영하고, 동시에 **외부동향 리서치**로 조정계수를 뽑아 행렬을 재조정한 시나리오를 만들고, **두 시나리오를 비교 시각화**한 뒤 **보고서 초안**까지 생성한다. 핵심은 이 **전 과정이 화면에서 단계별로 시각적으로 진행되는 것**.

## 1. 기술 스택

- Python 3.11+, **Streamlit** (단일 앱)
- pandas, numpy
- **plotly** (인터랙티브 차트)
- **SQLite (in-memory)** — "SQL로 데이터를 끌어온다"는 과정을 실제로 보여주기 위해 더미데이터를 SQLite에 적재하고 `SELECT`로 조회하는 흐름을 시연
- **외부 리서치 에이전트: LangGraph 기반** (`langgraph`, `langchain-anthropic`) — 검색 → 평가 → 재검색 루프 → 조정계수 추출의 **상태 그래프**로 구현
- **실제 web search** 사용 (Anthropic web search tool 또는 Tavily/검색 API 중 붙이기 쉬운 것). 검색 노드는 real 모드로 동작
- 보고서 생성: Anthropic API 호출
- 환경변수로 `ANTHROPIC_API_KEY`(+ 검색 API 키 필요시) 읽기. **키 없으면 데모가 끊기지 않게 목업 폴백** 유지 (보고서·검색 모두)

## 2. 화면 UX 흐름 (가장 중요)

1. 상단: 제목 + 짧은 설명. 입력 위젯 — **기준연도(기본 2026)**, **목표연도(기본 2031)**, (선택) **목표 필요인력**, **외부보정 on/off 토글**
2. `[예측 실행]` 버튼
3. 실행하면 **단계별 진행이 눈에 보이게**: `st.status` 또는 단계 카드 + 스피너로 각 단계가 순차 실행되고, **각 단계마다 중간 산출물(데이터 미리보기, 행렬, 트렌드 표, 차트)을 그 자리에서 펼쳐 보여줌**. 단계가 한 번에 끝나고 결과만 뜨면 안 됨 — **과정이 핵심**.
4. **내부 트랙**과 **외부 트랙**을 좌우 2컬럼으로 배치해 "병렬로 돈다"는 느낌을 시각적으로 연출 (실제 멀티스레딩은 불필요, 시각적 병렬이면 충분. 단계 사이에 짧은 `time.sleep`을 넣어 진행감을 줘도 됨)
5. 두 트랙이 끝나면 **수렴 단계**에서 비교 시각화 → 마지막에 **보고서 초안** 표시

## 3. 단계별 상세 스펙

### Stage 0 — 더미데이터 생성 (개인 단위 레코드)
- **개인 단위**로 생성: 컬럼 `사번, 직급, 근속연수, 부서, 기준연도, 다음연도_상태`
- 규모: 수천~1만 행 (포스코 가정, 파라미터로 조절 가능)
- **랜덤이 아니라 구조를 심을 것.** 데모 스토리가 차트에서 자연스럽게 튀어나오게:
  - 예: **차장급 이탈률을 의도적으로 높게** → 결과에서 "차장 구간 인력 누수(병목)"가 보이도록
  - 예: 사원→대리 승진은 활발, 대리→차장은 정체
- 생성 가정은 **함수 한 곳(`generate_dummy.py`)에 모으고, 가정값을 상단 상수로** 빼서 나중에 쉽게 바꾸게

### Stage 1 — 내부데이터 수집·전처리
- 생성한 더미를 **SQLite(in-memory)에 적재**
- 화면에 **실제 `SELECT` 쿼리문을 노출**하고 그 결과 DataFrame을 미리보기로 표시 ("SQL로 끌어온다"를 시연)
- pandas로 전처리: 상태 라벨링, 결측/이상치 정리(더미라 형식적이어도 됨), **전이 집계(GROUP BY 상태 → 다음상태 카운트)**
- 산출물: **전이 카운트 행렬**을 화면에 표로 표시

### Stage 2 — 마르코프 모델링 (내부 베이스라인)
- **상태 정의 (PoC 권장 7개):**
  `사원_0-2년, 사원_2년+, 대리_0-2년, 대리_2년+, 차장_0-2년, 차장_2년+, 이탈(흡수상태)`
  - 근속밴드를 상태에 넣어 **duration-aware(준마르코프)** 설계
- 전이행렬 P 추정 = 전이카운트를 행 기준 정규화
- **라플라스 평활** 적용 (카운트에 +0.5 후 정규화) — 0 관측 전이를 0%로 박제하지 않게. *코드에 주석으로 "실데이터에서 표본 적을 때 작동하는 장치"라고 명시*
- 투영: 인력벡터 `n(t)`에서 `n(t+1) = n(t) · P` 를 **기준연도→목표연도까지 연도별로 반복**
- **시각화:**
  - 전이행렬 P **히트맵**
  - 연도별 **총 인원 추이 라인차트**
  - **직급별 인원 추이**(스택/그룹 막대 또는 라인)
- 마르코프 가정 한계를 화면 하단에 **한 줄 캡션**으로 명시 (예: "무기억성 가정 — 실제 이탈은 근속·연령 영향, 본 모델은 근속밴드로 부분 보완")

### Stage 3 — 외부동향 리서치 에이전트 (LangGraph, 병렬 트랙)

**검색 → 평가 → (부족 시) 재검색 루프 → 조정계수 추출** 을 LangGraph 상태 그래프로 구현. 이 **검증 루프가 화면에 실시간으로 보이는 것**이 이 단계의 핵심 볼거리.

**그래프 구조:**
```
START → research → evaluate → [조건분기]
                                 ├─ 부족 & retry<2 → research (루프백)
                                 └─ 충분 또는 retry==2 → extract → END
```

**State (노드 간 전달):**
`keyword, search_results, eval_score, eval_feedback, retry_count(최대 2), coefficients`

**각 노드:**
- `research`: **실제 web search** 호출 (산업/직군 키워드로 최신 인력시장 동향 수집). 재검색 시엔 직전 `eval_feedback`을 반영해 쿼리를 보강
- `evaluate`: **진짜 평가** — "조정계수로 쓸 만한 정량/방향성 정보가 트렌드별로 확보됐는가"를 LLM이 0~100 점수 + 부족사유로 판정. *절대 "1회차는 무조건 부족" 같은 스크립트로 짜지 말 것 — 실제 점수에 따라 루프가 돌아야 정직한 데모가 됨* (평가 노드는 Haiku로 돌려 비용 절감 가능)
- `extract`: 충분해진 결과에서 **조정계수 매핑 테이블** 추출

**입력:** 산업/직군 키워드 (예: "철강·제조 인력시장", "AI 도입", "정년·신규채용 동향")

**검증 과정 시각화 (필수):**
- 각 노드 실행 시 Streamlit에 **실시간 로그 카드**로 진행 찍기:
  `🔍 검색 1회차 → 📊 평가 65점(부족: 정량 근거 약함) → 🔄 재검색 → 📊 평가 88점(충분) → ✅ 조정계수 추출`
- 평가 점수를 **게이지/막대**로, 재시도 이력을 **타임라인**으로 표시
- 앱 어딘가에 그래프 구조 다이어그램을 한 번 렌더 (`graph.get_graph().draw_mermaid()` 활용) — "이게 에이전트 구조다"를 한눈에

**산출물 1:** **트렌드 요약 카드** (트렌드명 + 한 줄 설명 + 방향성)
**산출물 2 (핵심):** **조정계수 매핑 테이블**을 명시적으로 표시. 블랙박스 금지.

| 외부 트렌드 | 영향 대상 전이 | 조정 |
|---|---|---|
| 노동시장 과열 | 대리 → 이탈 | +5%p |
| 사업 구조조정 시그널 | 사원 → 이탈 | +3%p |
| 처우 개선 정책 | 차장 잔류 | +4%p |

- 이 테이블은 **"가정(시나리오 레버)"로 포지셔닝** — 모델이 알아서 보정한 게 아니라 명시적 가정임을 강조

**폴백:** 검색/LLM 키가 없으면 canned 트렌드+점수 시퀀스로 동일 UI가 끝까지 돌게 (데모 중단 방지). 단 폴백임을 코드 주석에 명시.

### Stage 4 — 행렬 재조정 (외부보정 시나리오)
- Stage 3의 조정계수를 **P 사본에 적용** (해당 전이확률 ±조정)
- 적용 후 **각 행을 다시 정규화**(이탈 흡수상태 포함, 합=1 유지)
- 재조정된 P로 동일하게 연도별 투영 → **보정 시나리오 n'(t)** 생성
- 조정 전/후 P를 **나란히 히트맵**으로 보여줘서 "어디가 어떻게 바뀌었나" 가시화

### Stage 5 — 두 시나리오 비교 시각화 (수렴 단계)
- **Baseline(내부만)** vs **Adjusted(외부보정)** 를 한 화면에서 비교:
  - 총 인원 추이 **겹쳐 그리기**
  - 직급별 추이 비교
  - **델타(차이) 분해 뷰** — 어느 직급/어느 시점에서 갭이 벌어지는지를 별도 차트로. *이게 인사이트의 핵심이니 꼭 넣을 것*
- (선택) 목표 필요인력 입력 시: **필요인력 − 예측 잔존인력 = 갭** 을 계산해 "이 직군 N명 부족, 선제 대응 필요" 메시지 생성

### Stage 6 — 보고서 초안 생성
- 위 단계의 **구조화된 산출물(숫자·트렌드·갭)을 LLM에 넘겨** 한국어 컨설팅 보고서 초안 생성
  - 섹션: ① 요약 ② 내부 인력 전망(베이스라인) ③ 외부동향 ④ 시나리오 비교·갭 ⑤ 시사점·선제 제언
- API 없으면 **템플릿 기반 목업 초안**으로 폴백 (숫자는 실제 계산값을 채워넣어 그럴듯하게)
- 화면에 마크다운으로 렌더 + `[다운로드(.md)]` 버튼

## 4. 데이터 명세서 (필수 산출물)
- 더미라도 **"실제로는 어디서 오나"를 문서화**한 표를 앱 안의 별도 섹션(또는 `DATA_SPEC.md`)으로 포함:

  | 항목 | 실데이터 출처(가정) | 데모 더미 생성 방식 |
  |---|---|---|
  | 전이 카운트 | HR 시스템 인사이동 이력 테이블 | generate_dummy.py 가정값 |
  | 조정계수 | 외부 리서치 → 전문가 검토 | 매핑 테이블 상수 |

- 목적: PoC를 "장난감"이 아니라 **"실데이터 전환 설계도"**로 보이게 하기 위함

## 5. 파일 구조 (모듈 분리 — 교체 지점 명확히)
```
app.py                # Streamlit 메인 (UX 흐름·단계 오케스트레이션)
generate_dummy.py     # 더미 생성 (가정값 상수화) ← 실데이터 교체 지점
data_pipeline.py      # SQLite 적재·SQL 조회·pandas 전처리·전이 집계
markov.py             # 상태정의, P 추정, 라플라스 평활, 연도별 투영, 행렬 재조정
research_agent.py     # LangGraph 그래프(research→evaluate→재검색 루프→extract) + 실 web search + 조정계수 추출 + 목업 폴백
report.py             # 보고서 초안 생성 (real/mock)
viz.py                # plotly 차트 함수 모음
DATA_SPEC.md          # 데이터 명세서
README.md             # 실행법, 실데이터 교체 가이드
```

## 6. 교체 가능성(swap points) — README에 명시
- **더미 → 실데이터**: `generate_dummy.py` 대신 실제 HR 추출로 교체, `data_pipeline.py`의 SQL만 실제 테이블에 맞게 수정
- **에이전트 mock ↔ real**: 환경변수로 토글
- **상태 해상도**: `markov.py`의 상태 리스트만 바꾸면 확장(직급/근속밴드 추가)
- **고도화 로드맵 한 줄**: "데이터 규모 충분 시 계층적 풀링/베이지안(디리클레)으로 추정 안정화 가능" — README에 적어둘 것

## 7. 비기능 요구
- 한국어 UI
- API 키 없어도 **반드시 끝까지 실행되는** 목업 폴백 (데모 중 끊기면 안 됨)
- 각 단계에 짧은 진행 표시로 "과정이 도는" 체감 제공
- 차트는 plotly로 인터랙티브하게, 색/레이블 깔끔하게
- 코드에 핵심 가정(평활·조정계수·마르코프 한계)은 주석으로 설명

## 8. 먼저 할 일
구현 전에 **상태 정의·전이 허용 규칙·더미 가정값**을 한 번 표로 정리해 보여주고, 그다음 코드 작성. 막히면 가정은 합리적으로 정하고 진행하되 주석으로 표시할 것.
